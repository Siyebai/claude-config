## 如何使用这个库

### 写前 (Phase B 风格锚定)
1. 确定本章风格模式 (`novel mode generate <mode-id>`)
2. 打开对应作家的至少 1 个场景拆解
3. 读逐句拆解，内化这个作家的"感觉"
4. 关注"反向验证"部分——知道什么绝对不能写比知道什么应该写更重要

### 写中 (实时参考)
- 写完一个场景后，对照"全局技法模式提取"，检查你的场景是否使用了对应的技法
- 情感场景 → 检查是否用了该作家写情感的方式
- 对话场景 → 检查是否用了该作家写对话的方式

### 写后 (质量检测)
- `novel mode apply <mode-id> <chapter-file>` → 自动检测违规项
- `novel mode compare <mode-id> <chapter-file>` → 对比风格参数匹配度
- 重点修复 HIGH 和 CRITICAL 级别的违规

## 场景拆解格式说明

每个场景拆解包含:

```
原文: "引文"
技法: 具体的写作技法（短句/比喻/感官/对话/节奏...）
功能: 这个技法在叙事中完成什么任务
```

拆解完成后提取该场景的**全局技法模式**（6项），每一项标注:
- 表现: 在这个场景中如何体现
- 可复用模式: 其他场景如何套用这个技法

最后给出**反向验证**: 用"非此模式"重写同一场景，标注所有违反的规则。
