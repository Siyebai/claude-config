# 八维小说质量评分体系 v1.0

> 基于 LongStoryEval (ACL 2025) 8 维度/20 子维度 + Fabula Entropy Indexing + 网文质量清单 + 多Agent协同评审
> 目标：每章可量化、每 10 章全身体检、全书可比对

---

## 一、评分体系总览


### 评分体系设计原则

1. **数据驱动**: 8 个主维度源自 LongStoryEval 对 340K Goodreads 评论的分析
2. **可操作性**: 每个子维度有具体的 1-10 评分锚点（不是抽象的描述）
3. **权重校准**: 权重基于 LongStoryEval 论文中对最终评分影响程度的统计结果
4. **双重验证**: 主观维度 + 客观维度互补，防止"技术好但不好看"的偏差
5. **Fabula Entropy 集成**: ETC/EWC 分数直接映射到 ① 和 ④ 的评分中

## 二、八维评分详细指标

### 维度 ①：情节与结构 (Plot & Structure) — 权重 20%

> LongStoryEval 结论：情节是影响读者评分的**第一重要**客观维度

#### 子维度 1.1：连贯性与因果链 (Coherence & Causality)

| 评分 | 描述 | 锚定标准 |
|------|------|---------|
| 9-10 | 完美连贯 | 每章事件合理导向下一章；所有因果链可追溯；无跳跃式发展 |
| 7-8 | 基本连贯 | 主要因果链清晰；偶有依赖巧合（≤2 次/10章） |
| 5-6 | 部分断裂 | 存在 3-5 处逻辑跳跃；需要读者自行脑补动机 |
| 3-4 | 明显断裂 | 多处事件缺乏因果关系；角色行为不可解释 |
| 1-2 | 严重混乱 | 事件随机发生；前后矛盾；无法形成故事脊椎 |

**自动化检测**: ETC (过渡一致性熵) ≤ 2.0 → 对应 7+ 分；ETC ≥ 4.0 → 对应 3- 分

#### 子维度 1.2：节奏控制 (Pacing)

| 评分 | 描述 |
|------|------|
| 9-10 | 快慢档比例合理 (2:5:3)；每章有明确的情感目标实现；无拖沓或仓促感 |
| 7-8 | 基本合理；偶有 1-2 章节奏失衡但整体可控 |
| 5-6 | 节奏不稳；连续 3+ 章快档（疲劳）或慢档（拖沓） |
| 3-4 | 严重失衡；大段纯对话或大段纯描写 |
| 1-2 | 无节奏意识；读者情绪无起伏 |

**自动化检测**: Anti-Rush Brake 违规次数 + 情感曲线起伏频率

#### 子维度 1.3：惊喜与反转 (Surprise & Twists)

| 评分 | 描述 |
|------|------|
| 9-10 | 反转可预测但不可预见（事后合理化）；每 10 章至少 1 次实质性反转 |
| 7-8 | 有惊喜但部分可提前猜到；反转频率合理 |
| 5-6 | 反转生硬或完全可预测；依赖套路 |
| 3-4 | 靠意外维持悬念（角色突然变傻/信息隐瞒） |
| 1-2 | 无反转或反转完全不合理 |

#### 子维度 1.4：结局质量 (Ending Quality)

| 评分 | 描述 |
|------|------|
| 9-10 | 结局满足但不出乎主题逻辑；回收 ≥80% 伏笔；终场画面与开场对照 |
| 7-8 | 结局合理；回收 ≥60% 伏笔 |
| 5-6 | 结局匆忙或拖沓；部分伏笔未回收 |
| 3-4 | 结局不符合故事逻辑；大量伏笔未回收 |
| 1-2 | 烂尾（机械降神/强行圆满/主题背叛） |

### 维度 ②：角色塑造 (Characters) — 权重 18%

> LongStoryEval 结论：角色是**第二重要**的客观维度

#### 子维度 2.1：角色深度与复杂性 (Depth & Complexity)

| 评分 | 描述 |
|------|------|
| 9-10 | 每个主要角色有 Want/Need/Flaw/Ghost；Want≠Need 产生内在冲突；反派有合理 Justification |
| 7-8 | 主角有清晰欲望架构；反派有动机但不够复杂 |
| 5-6 | 角色有基本目标但缺乏内在矛盾；反派单薄 |
| 3-4 | 角色脸谱化；行动无深层动机 |
| 1-2 | 角色是情节的工具人；无任何个体性 |

**自动化检测**: 角色行为一致性验证 5 项检查的通过率

#### 子维度 2.2：角色成长弧线 (Character Arc)

| 评分 | 描述 |
|------|------|
| 9-10 | 每个主角的 Arc 有明确的起点→转折→终点；成长不是突然的而是阶梯式的 |
| 7-8 | 主角有明显成长；但在关键转折处略显跳跃 |
| 5-6 | 有成长但不够连贯；部分章节角色状态无推进 |
| 3-4 | 角色扁平（无变化）；或变化不合理（一天变圣人） |
| 1-2 | 角色状态混乱；前后矛盾 |

**自动化检测**: Arc 推进曲线的连续性检查 + 关键转折点标记

#### 子维度 2.3：角色关系真实性 (Relationship Authenticity)

| 评分 | 描述 |
|------|------|
| 9-10 | 关系有张力矩阵（状态/方向/转折点）；变化有铺垫有爆发有后果 |
| 7-8 | 主要关系有发展逻辑；但配角关系较单薄 |
| 5-6 | 关系发展依赖巧合或误会；缺乏真实互动 |
| 3-4 | 关系变化无铺垫；上章还恨这章就爱 |
| 1-2 | 所有关系服务于情节需要；无独立生命 |

#### 子维度 2.4：角色区分度 (Character Distinctiveness)

| 评分 | 描述 |
|------|------|
| 9-10 | 每个角色可通过对话/行为/决策逻辑单独识别；无两个角色可互换 |
| 7-8 | 主要角色区分清晰；部分配角可合并 |
| 5-6 | 有几个角色功能重叠；需要标签式区分（"戴眼镜的""高个子的"） |
| 3-4 | 多个角色说话/行为方式相同 |
| 1-2 | 所有角色是作者的分身 |

### 维度 ③：文笔与语言 (Writing & Language) — 权重 12%

> LongStoryEval 结论：文笔被频繁提及但对最终评分的影响小于情节和角色

#### 子维度 3.1：语言流畅度 (Fluency & Readability)

| 评分 | 描述 |
|------|------|
| 9-10 | 无错别字/语病/人称混乱；段落长度适中（200-500字）；对话正确分段 |
| 7-8 | 偶有错别字（≤1 个/章）；整体流畅 |
| 5-6 | 每章有 3-5 处语病；部分段落过长 |
| 3-4 | 频繁语病；人称混乱；需要猜测作者意图 |
| 1-2 | 严重影响阅读；需要重写 |

#### 子维度 3.2：叙事腔调与风格 (Voice & Style)

| 评分 | 描述 |
|------|------|
| 9-10 | 有清晰统一的叙事腔调；风格匹配题材预期；对话有角色辨识度 |
| 7-8 | 腔调基本统一；但偶有风格跳跃 |
| 5-6 | 风格不稳定；不同章节腔调不一致 |
| 3-4 | 无意识风格；模仿痕迹重 |
| 1-2 | AI 味极重；风格混乱 |

**自动化检测**: Style Curator Agent 的风格一致性评分

#### 子维度 3.3：描写密度与节奏感 (Description Balance)

| 评分 | 描述 |
|------|------|
| 9-10 | 描写密度匹配章节节奏目标；快档描写精简、慢档描写丰满 |
| 7-8 | 描写量基本合理；偶有描写过浓或过淡 |
| 5-6 | 描写与节奏脱节；高潮章节却有大段风景描写 |
| 3-4 | 描写严重不足（白水）或严重过度（堆砌） |
| 1-2 | 完全无视描写与节奏的关系 |

---

### 维度 ④：世界观与设定 (World-Building) — 权重 10%

> LongStoryEval 结论：世界观被频繁提及但对评分的影响权重较低 — 除非严重矛盾

#### 子维度 4.1：设定一致性与逻辑自洽 (Internal Consistency)

| 评分 | 描述 |
|------|------|
| 9-10 | 所有世界规则可追溯；R0-R3 规则优先级严格执行；违规有代价记录 |
| 7-8 | 核心规则一致；偶有次级规则前后偏差 |
| 5-6 | 存在 3+ 处规则矛盾；读者能注意到 |
| 3-4 | 规则随心所欲；为情节需要任意修改 |
| 1-2 | 无规则意识；世界是随便搭建的舞台 |

**自动化检测**: EWC (世界一致性熵) ≤ 1.5 → 对应 7+ 分；EWC ≥ 3.5 → 对应 3- 分

#### 子维度 4.2：世界深度利用 (Depth Utilization)

| 评分 | 描述 |
|------|------|
| 9-10 | 每 10 章至少 1 次世界规则的深度应用；主角利用规则解决困境 |
| 7-8 | 有利用但不频繁；更多是背景设定 |
| 5-6 | 世界设定是背景板；不参与情节 |
| 3-4 | 大量设定从未使用；只存在于设定文档中 |
| 1-2 | 设定与故事完全无关 |

#### 子维度 4.3：沉浸感与独特性和 original (Immersion & Originality)

| 评分 | 描述 |
|------|------|
| 9-10 | 世界有独立的"质感"；读者能想象生活在其中；有对标作品没有的独特设定 |
| 7-8 | 有原创元素；但整体在已有框架内 |
| 5-6 | 设定常见但描写有氛围感 |
| 3-4 | 设定即视感强；不同作品的缝合 |
| 1-2 | 完全照搬已有世界观 |

---

### 维度 ⑤：主题与思想深度 (Themes) — 权重 8%

#### 子维度 5.1：主题呈现的成熟度 (Theme Maturity)

| 评分 | 描述 |
|------|------|
| 9-10 | 主题通过故事自然呈现而非说教；有多个层次供不同读者解读 |
| 7-8 | 主题清晰但呈现方式偏直白 |
| 5-6 | 主题口号式出现；角色变成作者传声筒 |
| 3-4 | 想表达很多但什么都没表达清楚 |
| 1-2 | 无主题意识 |

#### 子维度 5.2：价值观冲突的构建 (Value Conflict Construction)

| 评分 | 描述 |
|------|------|
| 9-10 | 核心冲突是价值观冲突（不是简单的善恶对立）；敌对双方各有合理立场 |
| 7-8 | 有价值观冲突但不够深刻；配角立场较单一 |
| 5-6 | 冲突停留在利益/力量层面；缺乏价值内核 |
| 3-4 | 纯粹的善恶对立；反派为恶而恶 |
| 1-2 | 无冲突或冲突毫无意义 |

---

### 维度 ⑥：情感冲击 (Emotional Impact) — 权重 12%

> LongStoryEval 结论：情感冲击是**最重要的主观维度**

#### 子维度 6.1：情感起伏与多样性 (Emotional Range)

| 评分 | 描述 |
|------|------|
| 9-10 | 情感曲线有明确的高低起伏；连续 5 章同一情绪 = 0 次违规；读者经历多种情感（爽/燃/虐/暖/惊） |
| 7-8 | 情感有变化但高潮/低谷区分度不够 |
| 5-6 | 情感单一（全程高能或全程平淡） |
| 3-4 | 情感表达无效；严重依赖形容词而非场景 |
| 1-2 | 完全无法调动读者情绪 |

**自动化检测**: 情感曲线分析 + 情绪标签分布统计

#### 子维度 6.2：情感兑现的力度 (Emotional Payoff)

| 评分 | 描述 |
|------|------|
| 9-10 | 情感铺垫有兑现（埋下→引爆→后果）；读者在关键节点有明显情绪反应 |
| 7-8 | 主要情感线有兑现；但些许仓促 |
| 5-6 | 铺垫后未兑现（情感空炮）；或兑现不如预期 |
| 3-4 | 情感需求被无视 |
| 1-2 | 无任何有效情感兑现 |

#### 子维度 6.3：共情与代入感 (Empathy & Immersion)

| 评分 | 描述 |
|------|------|
| 9-10 | 读者关心主角的命运；主角的胜利让人喜悦、失败让人心痛 |
| 7-8 | 有共情但不深入；读者保持一定距离 |
| 5-6 | 主角的行为让人无法共情（太蠢/太完美/太讨厌） |
| 3-4 | 对主角的命运漠不关心 |
| 1-2 | 反感主角 |

### 维度 ⑦：阅读体验与成瘾性 (Enjoyment & Engagement) — 权重 12%

#### 子维度 7.1：钩子密度与翻页冲动 (Hook Density)

| 评分 | 描述 |
|------|------|
| 9-10 | 每章结尾有钩子（目标 100%）；钩子类型多样（信息钩/悬念钩/情感钩/反转钩） |
| 7-8 | ≥80% 章节有钩子；类型略有重复 |
| 5-6 | 50-80% 有钩子；部分章节自然结束无动力续读 |
| 3-4 | <50% 有钩子；读者可以随时放下 |
| 1-2 | 几乎无钩子 |

#### 子维度 7.2：爽点间隔与密度 (Payoff Density)

| 评分 | 描述 |
|------|------|
| 9-10 | 最大爽点间隔 ≤ 3 章；爽点类型多样（打脸/升级/揭秘/破镜/扬名） |
| 7-8 | 间隔 3-5 章；类型 ≥3 种 |
| 5-6 | 间隔 6-10 章；类型较单一 |
| 3-4 | 间隔 >10 章；爽点同质化严重 |
| 1-2 | 无爽点意识 |

#### 子维度 7.3：信息释放节奏 (Information Pacing)

| 评分 | 描述 |
|------|------|
| 9-10 | 信息释放有节奏（新信息/回溯/铺垫交织）；既不信息轰炸也不信息饥渴 |
| 7-8 | 基本合理；偶有信息过载或不足 |
| 5-6 | 信息释放无规划；要么全盘托出要么一直藏着 |
| 3-4 | 读者始终处于困惑状态（信息不足）或疲惫状态（信息过量） |
| 1-2 | 无任何信息管理意识 |

### 维度 ⑧：预期管理 (Expectation Fulfillment) — 权重 8%

#### 子维度 8.1：类型承诺兑现 (Genre Promise)

| 评分 | 描述 |
|------|------|
| 9-10 | 开头承诺的类型体验在全书兑现；不会从喜剧变悲剧（除非有铺垫） |
| 7-8 | 类型基本稳定；偶有偏向但可接受 |
| 5-6 | 类型摇摆；读者不知道下一段是什么风格 |
| 3-4 | 挂羊头卖狗肉（开头热血后面宫斗） |
| 1-2 | 完全背离类型承诺 |

#### 子维度 8.2：伏笔回收满足度 (Foreshadowing Payoff)

| 评分 | 描述 |
|------|------|
| 9-10 | ≥80% 伏笔回收；回收方式自然合理；读者能体验到"原来如此"的满足感 |
| 7-8 | ≥60% 回收；回收基本合理 |
| 5-6 | 40-60% 回收；部分伏笔被遗忘 |
| 3-4 | <40% 回收；大量未完成线索 |
| 1-2 | 几乎不回收伏笔 |

## 三、评分执行协议

### 3.1 三层评分节奏

```
每章评分 (Chapter Gate Score):
  □ ③ 文笔与语言 (Style Curator 执行)
  □ ⑥ 情感冲击 (Editor 执行)
  □ ⑦ 阅读体验 (Editor 执行)
  □ Gate 结果: passed (所有 ≥ 7.0) / warning (有 5.0-6.9) / failed (有 < 5.0)

每 10 章评分 (Milestone Health Check):
  全部 8 维度 20 子维度
  □ Continuity Guardian → ① 情节 / ④ 世界观
  □ Character Reviewer → ② 角色
  □ Style Curator → ③ 文笔
  □ Editor → ⑤ 主题 / ⑥ 情感 / ⑦ 体验 / ⑧ 预期
  → 生成 health_check_report.md

全书评分 (Final Quality Score):
  全部 8 维度 × 权重 → Q-Score
  □ 目标: ≥ 8.5/10
  □ 出版级: ≥ 9.0/10
  □ 可发布: ≥ 7.5/10
  □ 需修改: 6.0-7.4
  □ 需重写: < 6.0
```

### 3.2 评分公式

```
Q-Score = Σ (维度得分 × 维度权重)

其中:
  维度得分 = Σ (子维度得分 × 子维度权重) / Σ 子维度权重

Fabula Entropy 映射:
  ETC 评分 → 情节连贯性子维度 (1.1) 的基础分
    映射关系: ETC∈[1.0,2.0]→[10,7] / ETC∈[2.1,3.0]→[6,5] / ETC∈[3.1,4.0]→[4,3] / ETC∈[4.1,5.0]→[2,1]
  
  EWC 评分 → 世界观一致性子维度 (4.1) 的基础分
    映射关系: EWC∈[1.0,1.5]→[10,7] / EWC∈[1.6,2.5]→[6,5] / EWC∈[2.6,3.5]→[4,3] / EWC∈[3.6,5.0]→[2,1]
```

### 3.3 多Agent评审协议

```
评审流程 (每 10 章触发):

Step 1: 独立评审
  5 个 Agent 独立阅读 10 章内容，各自打分
  不允许 Agent 之间通信（防止群体思维）

Step 2: 交叉验证
  Continuity Guardian 的结果与 Editor 的结果对比
  同一维度评分差距 > 2 分 → 触发争议评审

Step 3: 争议处理
  争议维度由 Architect Agent 独立评审
  Architect 的评分作为最终裁定

Step 4: 报告生成
  合并所有 Agent 评分 → health_check_report.md
  包含: 各维度得分 / 趋势图 / 风险项 / 修复建议
```

### 3.4 评分锚定校准

为防止 Agent 评分漂移（aka 评分通胀/通缩），每 30 章进行一次校准：

```
校准流程:
  1. 从已评分的 30 章中随机抽取 5 章
  2. 各 Agent 重新评分（不看之前的分数）
  3. 对比新旧评分:
     平均偏差 ≤ 0.5 → 正常
     平均偏差 0.6-1.0 → 轻度漂移，标记关注
     平均偏差 > 1.0 → 严重漂移，重新校准评分标准
```

### 4.2 去 AI 味专项检查 (Deslop Check)

> 由 story-deslop skill 的"三遍法"执行

```
第一遍: 高频 AI 词汇扫描
  检查项: 目光/身形/冷笑/深深/不由得/似乎/仿佛/轻轻/微微/心中一惊/眼中闪过
  标准: 每 3000 字中 AI 高频词 ≤ 5 个
  违规: 标记并替换

第二遍: 弱化副词清理
  目标: 删除或替换 80% 的"很/非常/真的/特别/极其/无比/极其"

  检查: 意义膨胀词汇（"毁天灭地"/"惊天动地"/"不可思议"等）

通过标准: 三遍全部通过 → Deslop Pass → 才能进入正文门禁
```

## 七、质量修复协议

### 7.1 质量问题分级处理

| 级别 | 条件 | 处理 |
|------|------|------|
| **P0 阻塞** | Q-Score < 6.0 或 任一维度 < 4.0 | 立即停止写作 → 根因分析 → 重写问题章节 |
| **P1 紧急** | Q-Score 6.0-6.9 或 任一维度 < 5.0 | 完成当前章节后 → 修复问题 → 重评通过后继续 |
| **P2 关注** | Q-Score 7.0-7.4 或 连续下滑趋势 | 标记关注 → 下一 10 章补齐 |
| **P3 优化** | Q-Score ≥ 7.5 但有个别子维度 < 6.0 | 在后续章节中有意识优化 |

### 7.2 修复 SOP

```
P0/P1 修复流程:
  1. Architect Agent 分析根因
     Q: 是情节设计问题还是执行问题？
     Q: 是角色行为不合理还是铺垫不足？
     Q: 是单章问题还是系统性偏差？

  2. 生成修复方案
     情节问题 → 改大纲 / 补伏笔 / 拆章节
     角色问题 → 调整角色状态 / 补行为动机 / 改对话
     文笔问题 → 重写 / 风格校准 / 去 AI 味
     世界观问题 → 修正设定矛盾 / 记录 world_consistency_log

  3. 执行修复
     只修改有问题的部分，不做额外改动

  4. 重新评分
     修复后重评 → 确保问题解决 → 继续写作
```

## 八、与剧情推演引擎的集成

本评分体系是 `novel-plot-simulation-engine.md` 的姊妹系统：

| 引擎 | 职责 | 产出 |
|------|------|------|
| **剧情推演引擎** | Phase 0-7 全流程：种子→世界观→角色→情节→写作循环 | 故事内容 |

**集成点**:
- Phase 4-7 写作循环的「五步门禁」→ 嵌入本章的单章评分 (③⑥⑦)

- Fabula Entropy ETC/EWC → 自动映射到 ① 和 ④ 的评分
- Anti-Rush Brake → 映射到 ① 子维度 1.2 (节奏控制)
- 伏笔回收率 → 映射到 ⑧ 子维度 8.2 (伏笔回收满足度)
- Deslop Check → 映射到 ③ 子维度 3.2 (叙事腔调)
