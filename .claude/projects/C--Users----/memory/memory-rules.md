# 记忆系统规则

## 核心原则

**记忆存在文件里，不在会话缓存里。** 会话重启，脑内状态归零。文件永久。

## 文件分层

| 层级 | 文件 | 内容 | 更新频率 |
|------|------|------|----------|
| 索引 | MEMORY.md | 所有记忆文件指针 | 每次文件增删时 |
| 长期 | user-identity.md | 主公偏好、风格、技术栈 | 发现新模式时 |
| 长期 | agent-republic-project.md | 主项目状态、架构、依赖 | 项目有变更时 |
| 长期 | lessons-learned.md | 错误模式、根因、预防 | 每次犯错后 |
| 长期 | patterns.md | 反复出现的模式、有效策略 | 3次以上重复时 |
| 每日 | YYYY-MM-DD.md | 当日时间线、任务、决策、异常 | 每天 |

## 写入规则

1. **任务完成后立即记录** — 不等、不靠脑记
2. **决策记录理由** — 不只记"做了什么"，记"为什么这么做"
3. **异常记录根因** — 错误 → 根因 → 修复 → 预防，四要素完整
4. **精确时间戳** — 所有事件标注时间，便于回溯

## 提炼规则

每天/每几天执行一次提炼：

1. 通读最近的 daily files
2. 发现 ≥3 次重复的模式 → 提取到 patterns.md
3. 发现新用户偏好 → 更新 user-identity.md
4. 发现项目变更 → 更新 agent-republic-project.md
5. 过时信息 → 删除或标注 `~~已过时~~`
6. 提炼后 → 更新 MEMORY.md 索引

## 冲突检测

**规则：文件 > 会话记忆。文件是真相源。**

每次会话启动时交叉验证：
1. 读 MEMORY.md → 获取文件列表
2. 读今日 daily → 获取近期上下文
3. 对比当前环境（git status、运行进程、文件状态）
4. 不一致时 → **以文件为准**，标记差异报告主公核实

示例：
- daily 说 Agent Hub 运行在 18990，但 netstat 显示端口未监听 → 报告：daily 记录与实际不符，服务可能已停止
- MEMORY.md 引用了一个不存在的文件 → 清理索引

## 反遗忘协议

每次会话启动时，**强制执行**：
1. 加载 MEMORY.md（索引）
2. 加载今日 daily（如果存在）
3. 检查 last-session.json（上次会话断点）
4. 加载 user-identity.md（主公画像）
5. 短暂输出已知状态（1-2行），不啰嗦

此协议已写入 CLAUDE.md 和 soul.md，确保每次新会话自动执行。
