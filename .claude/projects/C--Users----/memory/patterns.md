# 长期模式

> 从 daily files 中提炼，≥3 次重复出现才记录。

## 工作模式

### 用户不在电脑旁时任务不能中断
- **出现**：5/17, 5/18, 5/19
- **模式**：权限弹窗、确认提示会阻断整个工作流
- **策略**：bypassPermissions + 所有操作直接执行，不等确认

### 会话重启丢失上下文
- **出现**：5/17, 5/18, 5/19
- **模式**：每次新会话需要重新查找资料
- **策略**：CLAUDE.md + 记忆文件系统 + Stop 钩子自动保存

### 规则文件膨胀
- **出现**：5/19（1次，待观察）
- **模式**：多个规则文件内容高度重叠
- **策略**：定期合并重叠规则，删除冗余文件

## 技术模式

### Windows Git Bash 兼容性
- 不支持 `grep -P`、`/dev/stdin`
- 中文路径可能编码异常
- 冒号 `:` 在文件名中被 git 转义为 `\357\200\272`
- **策略**：用专用工具替代 Bash 搜索，避免特殊字符文件名

### DeepSeek API 作为 Claude 后端
- Base URL: `https://api.deepseek.com/anthropic`
- 模型: deepseek-v4-pro (主), deepseek-v4-flash (子 agent)
- **策略**：通过 ANTHROPIC_BASE_URL 环境变量路由

## 小说系统模式 (2026-05-23)

### 记忆文件膨胀 → 定期整合
- **出现**：5/23 小说系统7个记忆文件互相交叉引用，大量内容重复
- **模式**：每次系统升级创建独立文档 → 同一内容在多个文件中各写一遍
- **策略**：7合1整合为 v5.0。保留详细参考文件但不让它们占据主索引。定期检查，≥5个同主题文件 → 启动整合

### 风格参数不是固定值 → 场景偏移
- **出现**：5/23 风格注入系统。每个大师的句长/对话比/感官配比是"默认值"
- **模式**：同一大师写对话 vs 写高潮 → 参数完全不同（余华默认句长10-18，高潮场景6-10）
- **策略**：主档案给默认值 + 场景深度矩阵给6类场景偏移量。调用时场景类型决定实际参数

### 冷热切换 — 正向热/负向冷
- **出现**：5/23 风格宪法 + 样板验证
- **模式**：正向情感高潮(胜利/重逢)用+2滚烫，负向情感高潮(死亡/告别)用-2极寒。越痛越冷
- **策略**：冷热切换五级温度模型(+2到-2)。负向高潮场景严格零情绪词+零评论+极短句

### 遮名可辨 — 七维语音差异
- **出现**：5/23 角色系统 + 样板验证
- **模式**：角色语音差异不只是"话多话少"。7个维度：句长偏好/词汇层级/句式习惯/节奏模式/口头禅/回避语言/对话策略
- **策略**：每角色一张语音卡。每场对话遮名重读。≥3维差异才算有效区分

### 工具链可执行化 → 质量内建
- **出现**：5/23 8个Node.js CLI工具构建
- **模式**：从"文档规范"变成"命令行检测" → 质量从靠自觉变成靠门禁
- **策略**：每章写完 → novel deslop scan + novel style lint + novel mode apply → PASS才进入下一章
