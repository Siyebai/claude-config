# 审计失败教训

## 错误
1. **扫文件名代读代码**: 见building-linkage目录名跳过，未读23,685行（战斗/赛马/武道会/市场订单簿/NPC引擎/56成就/粒子/天气）
2. **先入为主**: 预设"旧模块=人类游戏化遗留"用文件名验证，实际是完整游戏系统
3. **精确数字掩盖15%覆盖率**: 84,737行数字正确但实际只读了15-20%

## 补救
- 全面审计=读代码，不是扫文件名。地毯式=每个子目录入口文件都过。
- 先读再分类，不预设结论。
- 未读目录明确说"未读"，不带"已经覆盖"的暗示。
- 数字必须附覆盖率说明。"84,737行（覆盖20%）"而非"84,737行"。
