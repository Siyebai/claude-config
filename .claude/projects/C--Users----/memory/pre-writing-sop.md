# 写作前 SOP (Pre-Writing Standard Operating Procedure)

> Scribe 在写任何一章正文之前，**必须**按顺序走完此流程。
> 这不是事后检查清单，是写前引导。流程走完了，质量就内建了。

---

## Phase A: 上下文加载 (Smart State — 升级版)

> 整合 WenShape 事实摘要 + 对数距离衰减

### A1. 角色状态加载

```
加载文件: character_state.md
重点读取:
  ▸ 本章出场角色的当前状态 (位置/情绪/目标/关系/伤势)
  ▸ 上一章结束时每个出场角色的最后状态 → 本章必须从此状态出发
  ▸ 角色当前 Want 推进到哪一步了
```

### A2. 事实摘要检索 (NEW — WenShape 距离衰减)

```
加载文件: canon/facts.jsonl → 按章节距离衰减排序

检索策略:
  ▸ 本章章节号 N
  ▸ 对每个已记录事实，计算: weight = base_score × log(1 + 1/|N - fact_chapter|)
  ▸ 距离 ≤ 3 章的事实: 权重×1.0 (近期事实，关联最强)
  ▸ 距离 4-10 章的事实: 权重×0.7 (中期参考)
  ▸ 距离 > 10 章的重要世界观事实: 权重×0.5 (不丢弃，但降权)
  ▸ 取 Top-20 最相关事实注入上下文

关键: 越靠近当前章节的事实权重越高，远处的重要世界观事实不会被丢弃
```

### A3. 伏笔状态扫描

```
加载文件: plot_threads.md
检查:
  ▸ 本章应该回收哪些伏笔？(按计划回收章节筛选)
  ▸ 本章应该埋下哪些新伏笔？(按大纲预设)
  ▸ 未回收伏笔总数是否 > 已回收 × 1.5？→ 如是，优先回收
```

### A4. 世界规则刷新

```
加载文件: world_consistency_log.md
检查: 最近 5 章是否有规则变更？→ 本章必须遵守
```

## Phase B: 风格锚定 (Style Binding — NEW)

> 整合 AI-Novel-Assistant 写法引擎 + style_profile.md + 大师风格注入引擎 v1.0

### B0. 风格模式选择 (Style Mode Selection — NEW v2.0)

```
Architect 在规划阶段必须为本章指定风格模式:

  使用 novel mode list → 查看可用风格模式
  使用 novel mode generate <mode-id> → 生成 Scribe 可执行指令清单

风格模式分为:
  ▸ 单一大师模式: yu-hua / zhang-ailing / wang-xiaobo / jin-yong / liu-cixin / mo-yan / shen-congwen / lao-she / wang-zengqi
  ▸ 混合模式: novel mode blend <mode-a> <mode-b> <ratio> → 融合两种风格
  ▸ 自定义模式: 基于 style_profile.md 的四维参数，微调大师模式的参数

每个风格模式提供:
  ▸ 5 条可执行写作指令 (CRITICAL/HIGH/MEDIUM 分级)
  ▸ 6-8 条禁止模式 (写了就扣分)
  ▸ 场景权重配置 (对话/动作/心理/环境 各占多少)
  ▸ 目标参数范围 (句长/对话比/比喻密度/感官配比/叙事距离/段落长度)
  ▸ 叙述者声音描述 (语气/距离/词汇层级)

风格模式选择原则:
  ▸ 一本小说可以锁定一个主风格模式 (如全程余华模式)
  ▸ 不同章节可以在主模式基础上微调 (如高潮章节 +10% 张爱玲感官密度)
  ▸ 不同角色 POV 可以使用不同的风格模式变体
  ▸ 混合模式: base(70-80%) + modifier(20-30%)，保持主风格清晰
```

### B1. 风格金样本重读 (CRITICAL)

```
Scribe 必须执行:
  ▸ 打开 style_profile.md → 读"风格金样本"500字
  ▸ 如果本章指定了风格模式 → novel mode generate <mode-id> → 读 5 条写作指令 + 禁止模式
  ▸ 不是浏览，是朗读（内心里）
  ▸ 感受: 句子的长度节奏 / 词语的质感 / 对话的呼吸感
  ▸ 目标: 在脑海中建立风格音高 + 锁定禁用规则在意识中，再开始写
  ▸ 读完风格指令后在 Writing Authorization 中记录: style_mode: <mode-id>
```

### B2. 本章节奏意图确认

```
根据 beat_sheet.md 确认本章在节拍表中的位置:

  快档章节 (主线推进):
    ▸ 句子: 偏短，碎，直接
    ▸ 描写: 精简，紧扣动作
    ▸ 对话: 40-60%
    ▸ 结尾: 强钩子 (信息钩/悬念钩/反转钩)

  中档章节 (发展铺垫):
    ▸ 句子: 长短交替
    ▸ 描写: 适中
    ▸ 对话: 30-50%
    ▸ 结尾: 中钩子 (问题钩/情感钩)

  慢档章节 (消化期):
    ▸ 句子: 偏长，舒展
    ▸ 描写: 丰满，多感官
    ▸ 对话: 50-70%
    ▸ 结尾: 轻钩子 (情感钩/预示钩)
```

### B3. Anti-Rush 配额检查

```
检查: 上一章消耗了什么配额？
  ▸ 上一章消耗了 A → 本章必须是慢档或中档
  ▸ 上一章消耗了 B → 本章必须是中档
  ▸ 上一章消耗了 C → 本章必须是慢档
  ▸ 上一章无消耗 → 本章可以消耗 1 项
```

## Phase C: 场景澄清 (Scene Clarification — NEW)

> 整合 novel-writer /clarify 步骤

### C1. 强制追问 5 个问题

```
Scribe 在概要之前必须回答 (Architect 审查):

  Q1: 本章 POV 角色的 Want 是什么？(具体到这 3000 字里他想要什么)
  Q2: 阻止他的障碍是什么？(不能只是"敌人太强"，需要具体的障碍)
  Q3: 本章结束时，情况如何变化？(不能和开头一样)
  Q4: 读者在本章中获得什么新信息？(至少 1 条，不能是纯过渡章)
  Q5: 本章的情感弧线是什么？(读者开头什么情绪 → 结尾什么情绪)

如果任何一个问题回答不上来 → 本章不应该开始写 → 回到 Architect 重新规划
```

### C2. 角色语音卡匹配

```
确认本章出场的所有角色 → 打开每个人的语音卡
对比检查:
  ▸ 角色A和角色B的语音差异是否足够大？
  ▸ 本章对话中能否体现这些差异？
  ▸ 是否有角色的"禁用词"可能在本章被触发？→ 提前标记
```

## Phase D: 机械免疫预检 (Mechanical Immune Pre-Check — NEW)

> 整合 autonovel 纯正则免疫层 — 零 LLM 成本

### D1. 本章高危场景预标注

```
Scribe 在概要中标注:
  ▸ [情感场景] — 此场景禁止直接说出情绪词 ("他很愤怒" ❌)
  ▸ [打斗场景] — 此场景禁止长句/复杂比喻/过多描写
  ▸ [对话场景] — 此场景禁止角色交换信息式对话 (必须有冲突/潜台词)
  ▸ [揭秘场景] — 此场景禁止一次性全部揭示
  ▸ [过渡场景] — 此场景禁止超过 300 字
```

### D2. 禁止词预查

```
如果本章包含以下场景:
  情感场景 → 预查是否会出现: 心中一惊/心中涌起/不由得/似乎/仿佛
  打斗场景 → 预查是否会出现: 电光火石/瞬间/刹那间/快到极致
  对话场景 → 预查是否会出现: 冷声/淡淡道/微微一笑/沉声道
  结尾钩子 → 预查是否会出现: 他不知道的是…/然而等待着的是…

→ 在概要阶段就替换掉这些预判会出现的 AI 高频词
```

## Phase E: 概要写作 + 审查

### E1. 200-500 字概要

```
概要包含:
  1. 本章场景目标 (POV 角色要达成什么)
  2. 冲突类型 (人vs人/人vs社会/人vs自然/人vs自我)
  3. 场景序列 (本章内有几个场景，每个场景的功能)
  4. 情感弧线 (开头→中段→结尾 的情绪变化)
  5. 信息释放 (本章揭露什么新信息，铺垫什么)
  6. 结尾钩子 (具体是什么钩子类型)
  7. 节奏标注 (快/中/慢档，理由)
```

### E2. Architect 审查概要

```
Architect 检查:
  □ 场景目标是否与节拍表一致？
  □ 情感目标是否可达？
  □ 节奏标注是否与 Anti-Rush 配额兼容？
  □ 高危场景是否已预标注？
  □ 5 个澄清问题是否全部有答案？
```

## Phase F: 写作授权 (Write Authorization)

```
所有 Phase A-E 通过 → Architect 签发写作授权 → Scribe 开始写正文

写作授权记录:
  chapter: N
  style_mode: yu-hua         # 本章风格模式 (NEW v2.0)
  style_bound: true          # 已加载风格金样本 + 风格模式指令
  voice_cards_loaded: []     # 已加载的角色语音卡列表
  scene_goals_verified: true # 场景目标已审查
  anti_rush_clear: true      # 节奏配额已检查
  clarify_answers: 5/5       # 5 个澄清问题已回答
  mechanical_precheck: true  # 机械免疫预检通过

→ Scribe: 现在开始写正文。严格按照 Writing Commandments + 风格模式指令执行。
```

## 写作中实时约束

### 四类场景写作指令 (← AI_NovelGenerator 4.7k⭐)

**场景类型判断**: 落笔前先判断本章每个场景属于哪一类，按对应指令执行。

```
┌──────────────────────────────────────────────────────────┐
│ 对话场景                                                  │
│   ☐ 潜台词冲突 — 角色说的 ≠ 角色真正的意思                    │
│   ☐ 不对称对话长度 — 权力关系通过话量体现                      │
│   ☐ 对话即行动 — 每句话推动剧情/揭示角色/制造冲突               │
│   ☐ 避免纯信息交换式对话 — 即使是"解释"，也要在冲突中解释         │
│                                                          │
│ 动作场景                                                  │
│   ☐ 三种感官描写 (视觉+听觉+触觉) — 不是只有"看"               │
│   ☐ 短句加速 + 比喻减速 — 节奏要有变化                        │
│   ☐ 动作揭示角色特质 — 不是描述他是怎样的人，是展示他做什么        │
│   ☐ 禁止长句/复杂比喻/过多静态描写                             │
│                                                          │
│ 心理场景                                                  │
│   ☐ 认知失调的行为外显 — 内心矛盾 → 身体反应                   │
│   ☐ 隐喻系统连接世界观符号 — 角色的困惑用世界的意象来表达          │
│   ☐ 价值天平描写 — 角色在两类价值间摇摆的具体感受                 │
│   ☐ 禁止直接说出情绪词 ("他很愤怒" ❌ → "他下颌收紧，指节发白" ✅) │
│                                                          │
│ 环境场景                                                  │
│   ☐ 空间透视变化 — 宏观全景 → 异常焦点 → 角色反应              │
│   ☐ 非常规感官组合 — "听见阳光的重量" 式通感                   │
│   ☐ 环境反映心理 — 景语即情语，不是客观描述而是主观感受            │
│   ☐ 禁止纯风景描写 — 每个环境细节必须有叙事功能                  │
└──────────────────────────────────────────────────────────┘
```

### 写作中实时检查

```
每写完一个场景:
  □ 场景类型是否正确？(对话/动作/心理/环境)
  □ POV 是否一致？
  □ 是否有至少 3 种感官？
  □ 对话是否区分角色语音？

每写完 500 字:
  □ 是否有 filter words (saw/heard/felt/noticed) → 立即删改
  □ 是否有弱化副词 (very/really/quite) → 立即替换
  □ 连续 3 句以同一主语开头？→ 立即变化
  □ 是否有 AI 高频词？→ 立即替换 (参考 deslop-scanner)
```

### 角色状态更新检查 (← AI_NovelGenerator 五点追踪)

```
每章写完后更新 character_state.md:
  □ 物品变化？(获得/遗失/损坏)
  □ 能力变化？(升级/解锁/削弱)
  □ 身体&心理状态变化？
  □ 关系网变化？(新增/断裂/深化)
  □ 触发&加深事件？(本章有什么影响了角色)
```
