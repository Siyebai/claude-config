## 二、Phase 0: 种子提炼引擎

### 2.1 五要素深度萃取

不是简单问答，而是**压力测试式萃取**：

```
用户输入: "我想写一个废柴炼丹师逆袭的故事"

萃取流程:
  Step 1: 核心情感锚定
    Q: 这个故事的"一句话情绪"是什么？
    Q: 读者合上书后应该感受到什么？（爽/燃/虐/暖/惊）
    → 输出: emotional_core.md

  Step 2: 欲望-障碍-代价三角
    主角想要什么？（Want）
    为什么现在得不到？（Obstacle）
    得到后要付出什么？（Cost）
    他真正需要的是什么？（Need — 通常与Want相反）
    → 输出: desire_triangle.md

  Step 3: 逆推锚点
    先确定三个画面：
      P_END:   最终画面（最后一章的具体场景）
      P_MID:   中点画面（第50%处的转折场景）
      P_OPEN:  开场画面（第1%处的初始场景）
    → 这三个画面构成故事脊椎，后续所有大纲围绕它们展开

  Step 4: 五层追问（逆推大纲法）
    核心冲突 → 势力图谱 → 胜利路径 → 成长阶梯 → 情感基石
    每层衍生 3 个子问题，形成 15 节点剧情树
    → 输出: reverse_outline_tree.md

  Step 5: 五要素确认
    目标读者 / 写作风格 / 核心禁区 / 自动化等级 / 目标规模
    → 写入 idea_seed.md（宪法文件，后续不许改）
```

### 2.2 种子质量校验

完成 Phase 0 后，必须通过以下检查才能进入 Phase 1：

| 检查项 | 通过标准 | 失败处理 |
|--------|---------|---------|
| 情感锚唯一性 | 能用一句话说出故事的情感承诺 | 重新萃取，直到清晰 |
| 三角完整性 | Want/Obstacle/Cost/Need 全部非空且互相矛盾 | 重新设计三角 |
| 逆推三画面 | P_END/P_MID/P_OPEN 可视觉化、有情感冲击力 | 换画面，不要换故事 |
| 原创性检查 | 不与对标作品的 signature setup 重合 | 修改核心设定 |

## 三、Phase 1: 世界观引擎

### 3.1 三维世界建模

```
物理维度:         社会维度:          隐喻维度:
  地理/气候         阶级/权力结构       核心主题
  资源分布          规则/禁忌          价值观冲突
  技术/魔法体系      文化/习俗          哲学命题
  时间线            势力版图           象征体系
```

### 3.2 世界观规则验证

每条世界设定必须通过三问：

```
Q1: 这条规则会产生什么故事可能性？（至少 3 个）
Q2: 这条规则会不会在某处自相矛盾？
Q3: 主角能利用这条规则做什么？（不能只是背景板）
```

### 3.3 世界一致性约束

```
规则优先级:
  R0 (不可违背): 物理/魔法基本法则
  R1 (极难违背): 世界级制度/契约
  R2 (违背有代价): 社会规范/势力规则
  R3 (可违背): 个人约定/临时协议

每章违反规则的行为必须在 world_consistency_log.md 中记录代价
```

## 四、Phase 2: 角色引擎

### 4.1 欲望架构 (Character Want-Need Architecture)

```
主角:
  Want (表层欲望):   ________________________________
  Need (深层需求):    ________________________________
  Flaw (核心缺陷):    ________________________________
  Ghost (创伤来源):   ________________________________
  Arc (成长弧线):    ________________________________

反派:
  Want:              ________________________________
  Need:              ________________________________
  Justification:     为什么他认为自己是对的？（不能是纯恶）
  Mirror:            反派和主角的镜像关系是什么？

配角功能位:
  Ally (盟友):       催化主角成长 / 反映主角变化
  Mentor (导师):     提供智慧 / 可能死亡激发主角
  Shadow (影子):     与主角相似但选择不同道路
  Foil (对比):       衬托主角特质的对立面
  Shapeshifter (变量): 立场不可预测，制造悬念
```

### 4.2 角色关系张力矩阵

```
        主角  反派  Ally  Mentor  Shadow  Love
主角     -    冲突   信任   依赖    竞争    吸引
反派    冲突   -     仇恨   蔑视    利用    觊觎
Ally    信任  仇恨    -     尊重    怀疑    暗恋
Mentor  依赖  蔑视   尊重    -      担忧    无
Shadow  竞争  利用   怀疑   担忧    -      嫉妒
Love    吸引  觊觎   暗恋   无     嫉妒    -

标记每个关系的:
  ▸ 当前状态 (0% → 100%)
  ▸ 变化方向 (升温/降温/逆转)
  ▸ 关键转折点（具体章节）
```

### 4.3 角色行为一致性验证

每章完成后，对每个出场角色执行：

```
□ 这个角色的行动是否符合其 Want？（不符合需有外部压迫）
□ 这个角色的反应是否暴露其 Flaw？（完美反应 = 角色失真）
□ 这个角色的对话是否匹配其教育背景/身份/情绪状态？
□ 这个角色在本章中是否有弧线推进（哪怕微小）？
□ 与上一章相比，角色状态是否有逻辑变化？
```

## 五、Phase 3: 情节引擎（核心）

### 5.1 逆推大纲法（完整版）

```
Step 1 — 锚定终局（必须先做）
  用 500 字具体描写最终画面：
    谁在场？什么场景？什么情绪？什么对话？
    这个画面为什么能给整个故事画上句号？

Step 2 — 逆推卷高潮
  全文 N 卷，每卷 1 个高潮点
  从终局逆推 → 每卷高潮是什么 → 连接各卷的因果链

Step 3 — 逆推章节
  每章 1 个场景目标（Scene Goal）
  场景目标 = 主角想在本章达成什么 + 实际达成了什么

Step 4 — 生成剧情树
  从每个卷高潮衍生 3 条路径：
    路径 A: 最优解（最自然、最有张力）
    路径 B: 颠覆解（挑战读者预期但合理的）
    路径 C: 极端解（不推荐但有启发的）
```

### 5.2 Save the Cat 节拍计算器

输入总章节数，自动计算每个节拍对应的章节位置：

```
总章节数: ____ 章

| # | 节拍 | 位置 | 对应章节 | 章节目标 | 情感目标 |
|---|------|------|---------|---------|---------|
| 1 | 开场画面 | 1% | 第_章 | 建立日常+缺陷展示 | 好奇/共鸣 |
| 2 | 主题陈述 | 5% | 第_章 | 暗示主题的对话/事件 | 若有所思 |
| 3 | 铺垫 | 1-10% | 第_-_章 | 展示主角六个面 | 理解/投入 |
| 4 | 催化剂 | 10% | 第_章 | 打破日常的事件 | 震惊/期待 |
| 5 | 犹豫 | 10-20% | 第_-_章 | 主角抗拒改变 | 紧张/担忧 |
| 6 | 第二幕开启 | 20% | 第_章 | 跨过门槛 | 兴奋/好奇 |
| 7 | B故事 | 22% | 第_章 | 副线/爱情线启动 | 温暖/期待 |
| 8 | 娱乐游戏 | 20-55% | 第_-_章 | 承诺的"好看"部分 | 爽/燃/趣 |
| 9 | 中点 | 55% | 第_章 | 假胜利或假失败 | 震撼/转折 |
| 10| 反派逼近 | 55-75% | 第_-_章 | 压力升级 | 焦虑/压迫 |
| 11| 一无所有 | 75% | 第_章 | 最黑暗时刻 | 绝望/心碎 |
| 12| 灵魂黑夜 | 75-85% | 第_-_章 | 反思+觉醒 | 感动/希望 |
| 13| 第三幕开启 | 85% | 第_章 | 新方案/最后一搏 | 热血/决意 |
| 14| 结局 | 85-99% | 第_-_章 | 最终对决 | 酣畅/满足 |
| 15| 终场画面 | 99% | 第_章 | 与开场对照 | 回味/圆满 |

每章必须标注:
  ▸ 场景目标 (Scene Goal)
  ▸ 情感目标 (Emotional Goal)
  ▸ 冲突类型 (人vs人 / 人vs社会 / 人vs自然 / 人vs自我)
  ▸ 信息释放量 (新信息/回溯/铺垫)
```

### 5.3 分支推演模拟器

```
│   中期后果 (5-10章后):             │
│   长期后果 (20+章后):              │
│   对 Want 的影响:                  │
│   对 Need 的影响:                  │
│   对其他角色的影响:                 │
│   张力评分 (1-10):                 │
│   惊喜评分 (1-10):                 │
├────────────────────────────────────┤
│ 选项 B: [具体行动]                  │
│   (同上结构)                       │
├────────────────────────────────────┤

├────────────────────────────────────┤

│ 风险: 需在第_章设置伏笔回收         │
└────────────────────────────────────┘
```

### 5.4 Fabula Entropy 一致性度量

基于 Georgia Tech 的 Fabula Entropy Indexing 框架，每卷/每10章运行：

```
ETC (过渡一致性熵):
  对每对相邻章节提问：
    Q: "第N章的事件是否合理导向第N+1章？"
    评分: 1(非常合理) - 5(完全不连贯)
  目标: 平均分 ≤ 2.0

EWC (世界一致性熵):
  对随机采样的 10 个世界事实提问：
    Q: "事实X在第1章和第N章是否一致？"
    评分: 1(完全一致) - 5(严重矛盾)
  目标: 平均分 ≤ 1.5

检测算法:
  1. 从每章提取关键断言（character_state / world_fact / plot_event）
  2. 配对比较同类型断言
  3. 矛盾 → 标记 type (timeline_break / character_jump / world_contradiction / plot_hole)
  4. 生成 consistency_report.md
```

### 5.5 剧情加速制动器 (Anti-Rush Brake)

```
每章配额检查:
  □ A 主线矛盾实质性推进？（是=消耗配额）
  □ B 主要关系决定性升级？（是=消耗配额）
  □ C 核心秘密完整揭露？（是=消耗配额）

配额规则:
  每章至多消耗 1 项
  消耗 A → 下一章必须是慢档或中档（B/C 不得触发）
  消耗 B → 下一章必须是中档
  消耗 C → 下一章必须是慢档（需要消化期）

违规处理: 门禁强制失败 → Beat Sheet 重写 → 拆分事件到多章
```

## 六、Phase 4-7: 写作-门禁循环

### 6.1 单章写作 SOP

```
写作前（必须执行）:
  □ 读 character_state.md（所有出场角色当前状态）
  □ 读 plot_threads.md（待回收伏笔 + 已埋线索）
  □ 读 world_consistency_log.md（最近的世界规则变更）
  □ 确认本章的场景目标和情感目标
  □ 确认本章在节拍表中的位置

写作中:
  1. 200-500 字概要（Scene Outline）
  2. 概要审查（是否符合场景目标）
  3. 3000-5000 字正文（按 story-long-write 方法论）
  4. 自查: 情绪是否兑现 / 冲突是否有效 / 对话是否区分角色

写作后（五步门禁 — 不可跳过）:
  Step 1: 更新记忆
    ▸ character_state.md — 更新所有出场角色状态
    ▸ plot_threads.md — 记录新埋伏笔 + 标记已回收伏笔
    ▸ world_consistency_log.md — 记录任何世界规则变更
    ▸ synopsis.md — 追加本章 200 字摘要

  Step 2: 一致性检查
    ▸ 主角位置/状态连续性（不能瞬移）
    ▸ 时间线逻辑（前一天晚上的伤第二天不能消失）
    ▸ 角色关系状态正确（上章还恨的人这章不能爱）
    ▸ 世界规则一致性（魔法冷却时间/技能限制）

  Step 3: 风格校准
    ▸ 叙事腔调是否与 style_guide.md 一致
    ▸ 对话风格是否区分角色
    ▸ 描写密度是否匹配题材预期

  Step 4: 校稿
    ▸ 错别字 / 语病 / 人称混乱
    ▸ 段落过长 / 对话不分段
    ▸ 不得出现 [说明] /（注：）/ TODO / 写作分析

  Step 5: 门禁检查
    ▸ gate_result.json: {
        "passed": true/false,
        "memory_updated": true/false,
        "consistency_score": 0-10,
        "style_score": 0-10,
        "proofread": true/false,
        "anti_rush": true/false,
        "issues": []
      }
    ▸ passed=false → 唯一操作: /修复本章
```

### 6.2 三档节奏控制

```
快档章节 (主线推进): 配额消耗 = 1项核心推进
  情感密度: 高
  描写密度: 中低
  对话比例: 40-60%
  信息释放: 大量新信息

中档章节 (发展铺垫): 配额消耗 = 0项
  情感密度: 中
  描写密度: 中
  对话比例: 30-50%
  信息释放: 中等，侧重线索铺设

慢档章节 (消化期): 配额消耗 = 0项
  情感密度: 中高（通过关系/角色深度补偿）
  描写密度: 中高
  对话比例: 50-70%
  信息释放: 少量，侧重情感和角色关系
```

## 七、质量门禁矩阵

每 10 章触发一次**全面体检**（由 Continuity Guardian + Editor + Style Curator 三Agent 执行）

```
体检项目:
  □ 角色弧线健康度（每个主角的 Arc 推进是否符合计划）
  □ 伏笔回收率（已埋伏笔中已回收的比例，目标 > 60%）
  □ 节奏健康度（快/中/慢档比例是否合理，目标 2:5:3）
  □ 情感曲线（读者情绪是否有起伏，连续 5 章同一情绪 = 问题）
  □ 钩子密度（每章结尾是否有钩子，目标 100%）
  □ 爽点间隔（最大间隔不超过 3 章）
  □ 世界观深度利用（每 10 章至少 1 次世界规则的深度应用）
  □ 配角存在感（每个功能位配角每 10 章至少出场 1 次）

体检报告: health_check_report.md
  通过: 继续写作
  警告: 标记风险项，继续写作但下个 10 章需修复
  失败: 暂停写作，先修复风险项
```
